# aws-rds-postgres

Stands up an Amazon RDS for PostgreSQL instance with KMS-encrypted storage, IAM database authentication, master credentials in AWS Secrets Manager, and 0–5 read replicas behind an RDS reader endpoint.

## What it provisions

- RDS PostgreSQL instance with the chosen engine version, instance class, and storage
- 0–5 asynchronous read replicas behind a reader endpoint
- DB parameter group with `rds.force_ssl=1` and `pg_stat_statements`
- DB subnet group spanning the connected VPC's private subnets
- Security group restricting access to the VPC
- Master credentials autogenerated and stored in AWS Secrets Manager (or pulled from `master_password` override)
- IAM database authentication enabled by default
- Enhanced monitoring + optional Performance Insights
- Automated backups with configurable retention

## Connections

- `aws_authentication: aws-iam-role` — env-default, supplies the provisioning role
- `vpc: aws-vpc` — canvas-linked, places the database into private subnets

## Outputs

- `database: aws-rds-postgres` — endpoint, port, database name, master username, Secrets Manager secret ARN, and a set of bindable IAM policies. Downstream apps (e.g. `aws-app`) bind those policies via IRSA and pull credentials from Secrets Manager at runtime.

## Configuration highlights

- **`database_name`** — Initial database name. Marked immutable; cannot be renamed after creation.
- **`username`** — Master account name. Marked immutable. Workloads should use IAM auth, not this account.
- **`instance_class`** — Graviton (`db.t4g`, `db.r6g`) is the cheaper default. Resize is allowed but causes a restart.
- **`multi_az`** — Synchronous standby in a second AZ. Doubles cost; required for production-grade availability.
- **`storage_encrypted`** — KMS-encrypted storage at rest. Marked immutable; cannot be toggled after creation.
- **`deletion_protection`** — Blocks accidental deletion via API/console. Default on.

See `massdriver.yaml` for the full param surface.

## Compliance

The bundle ships a `.checkov.yml` skip list containing only false positives plus the project-wide no-CMK policy. Production environments (`md-target` matching `^(prod|prd|production)$`) hard-fail on Checkov findings; lower environments surface findings as warnings.
